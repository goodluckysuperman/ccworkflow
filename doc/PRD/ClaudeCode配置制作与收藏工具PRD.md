# 1. 需求名称

- Claude Code 配置制作、收藏与安装工具（一期）

# 2. 目标

1. 提供一个本地可执行程序 `.exe`。
2. 用户双击 `.exe` 后，系统自动打开浏览器中的本地页面。
3. 用户可在页面中制作、AI 生成、编辑、保存、检索 Claude Code 配置。
4. 用户可将收藏的配置自动安装到指定项目，或安装到用户全局目录。
5. 工具内置安装能力，用户不需要手动复制配置文本或手动执行安装命令。
6. 用户可卸载某个已安装的配置包。
7. 工具默认将收藏保存在 `D:\Programming_tools\.ccworkflow`。
8. 用户可在设置中修改收藏保存地址。
9. 用户修改收藏根目录时，系统自动迁移旧数据。

# 3. 背景

1. Claude Code 的 skills、hooks、MCP 配置格式不同。
2. 用户当前主要痛点是制作成本高、跨项目复用低效、人工粘贴容易出错。
3. 用户不再需要“一期读取和实时同步本地项目配置”的能力。
4. 一期优先解决“制作、收藏、自动安装、卸载”四件事。

# 4. 范围内

1. 支持的配置类型仅限：
   1. skills
   2. hooks
   3. MCP
2. 支持两种制作方式：
   1. 表单制作
   2. AI 描述生成草稿
3. 支持将多个对象保存为一个配置包。
4. 配置包内允许同时包含多种对象类型。
5. 支持查看、编辑、删除、复制配置包。
6. 支持按配置类型分类查看收藏。
7. 支持按名称、关键字、标签检索配置包。
8. 支持项目级安装。
9. 支持用户全局安装。
10. 支持自动创建缺失的目标文件或目标目录。
11. 支持对已有目标文件执行合并安装。
12. 同名对象冲突时，支持提示用户选择处理方式。
13. 支持卸载某个已安装的配置包。
14. 支持记录每次安装结果，用于后续卸载。
15. 支持在设置中修改收藏根目录。
16. 支持本地保存配置包元信息。
17. 支持保存附加脚本并在安装时一并落地。

# 5. 范围外

1. 不监听项目配置变化。
2. 不自动同步外部对项目配置文件的手动改动。
3. 不支持 subagents。
4. 不支持 rules / `CLAUDE.md` 生成与管理。
5. 不支持云同步。
6. 不支持多人协作。
7. 不支持在线分享平台、远程仓库拉取。
8. 不支持一键回滚任意历史安装记录。
9. 不支持恢复整个项目到“安装前完整状态”。
10. 不支持自动处理用户手工改坏后的复杂冲突。

# 6. 用户角色

1. 本地单用户。
2. 主要用户是需要频繁编写和复用 Claude Code 配置的开发者。
3. 用户运行环境默认是 Windows。
4. 用户具备选择项目目录和理解安装结果的基本能力。

# 7. 核心流程

## 7.1 表单制作流程

1. 用户进入“新建配置包”页面。
2. 用户填写配置包基础信息：
   1. 名称
   2. 简介
   3. 标签
3. 用户向配置包中添加一个或多个对象。
4. 每个对象必须选择类型：
   1. skill
   2. hook
   3. MCP
5. 用户通过表单填写对象的公共字段。
6. 用户在正文编辑区补充或修改最终配置文本。
7. 系统实时预览最终配置文本。
8. 用户保存配置包。
9. 系统进行校验。
10. 校验通过后写入收藏目录。

## 7.2 AI 生成流程

1. 用户进入“AI 生成”页面。
2. 用户填写配置包基础信息。
3. 用户输入自然语言描述。
4. 用户指定要生成的对象类型或对象组合。
5. 系统调用本机 Claude Code 生成配置草稿。
6. 系统展示生成结果，不得自动保存。
7. 用户可继续编辑草稿。
8. 用户确认后手动保存配置包。

## 7.3 收藏与检索流程

1. 用户进入配置包列表页。
2. 用户可按类型查看。
3. 用户可按关键字搜索。
4. 用户可按标签筛选。
5. 用户打开任一配置包后，可查看元信息、对象列表、配置文本、附加脚本信息。

## 7.4 安装流程

1. 用户打开配置包详情。
2. 用户选择安装范围：
   1. 项目级安装
   2. 用户全局安装
3. 用户选择安装目标。
4. 选择项目级安装时，用户必须输入项目根目录。
5. 选择用户全局安装时，系统直接使用用户全局标准目录。
6. 系统展示将要写入的目标路径和冲突检查结果。
7. 系统执行安装前校验。
8. 若目标文件不存在，系统自动创建。
9. 若目标文件已存在，系统按类型执行合并。
10. 若存在同名 skill、hook 或 MCP，系统提示用户选择：
    1. 替换
    2. 跳过
    3. 取消本次安装
11. 系统写入配置内容。
12. 系统复制附加脚本。
13. 系统记录本次安装结果。
14. 系统展示安装成功或失败结果。

## 7.5 卸载流程

1. 用户进入已安装记录列表。
2. 用户选择某个已安装配置包。
3. 系统展示该配置包曾写入的内容范围。
4. 用户确认卸载。
5. 系统仅移除该配置包安装进去的内容。
6. 系统移除该配置包安装进去的附加脚本。
7. 系统更新安装记录状态。
8. 卸载成功后提示结果。

## 7.6 设置流程

1. 用户进入设置页。
2. 用户查看当前收藏根目录。
3. 用户修改收藏根目录。
4. 系统校验新路径是否可用。
5. 用户确认迁移旧数据。
6. 系统执行旧数据迁移。
7. 迁移成功后保存设置。
8. 迁移失败时不得切换到不完整状态。

# 8. 关键规则

1. 一期仅支持 `skills`、`hooks`、`MCP` 三类对象。
2. 一个配置包可包含多个对象。
3. 一个配置包可包含多种对象类型。
4. AI 生成结果必须先展示为草稿。
5. AI 生成结果未经用户确认不得自动保存。
6. 工具必须支持项目级安装与用户全局安装。
7. 项目级安装时，用户必须输入项目根目录。
8. 用户全局安装时，系统不得要求输入项目根目录。
9. 目标文件不存在时，系统必须自动创建。
10. 目标文件已存在时，系统必须执行合并，而不是整文件覆盖。
11. 发生同名对象冲突时，系统必须提示用户选择替换、跳过或取消安装。
12. hooks 与 MCP 一期只做“语法 + 必填字段”校验。
13. 配置正文为空时不得保存。
14. 对象类型缺失时不得保存。
15. 保存名称为空时不得保存。
16. 保存配置包时必须保存元信息。
17. 配置包元信息至少包括：
    1. 名称
    2. 简介
    3. 标签
    4. 创建时间
    5. 更新时间
    6. 包含的对象类型
18. 工具必须按配置类型提供列表分类入口。
19. 单一类型配置包必须保存到对应分类目录。
20. 多类型配置包必须保存到 `mixed` 分类目录。
21. 手工编辑配置文本时，预览内容必须同步更新。
22. skills 项目级安装路径默认是 `<project>/.claude/skills/<skill_name>/SKILL.md`。
23. hooks 项目级安装路径默认是 `<project>/.claude/settings.json`。
24. MCP 项目级安装路径默认是 `<project>/.mcp.json`。
25. skills 全局安装路径默认是 `~/.claude/skills/<skill_name>/SKILL.md`。
26. hooks 全局安装路径默认是 `~/.claude/settings.json`。
27. MCP 全局安装路径默认是 `~/.claude.json` 的全局 `mcpServers` 配置。
28. 附加脚本的项目级安装路径默认是 `<project>/.claude/scripts/`。
29. 附加脚本的全局安装路径默认是 `~/.claude/scripts/`。
30. 工具内置安装能力，用户不需要手动复制文本或手动执行安装命令。
31. 保存配置包不会自动触发安装。
32. 每次安装都必须生成安装记录。
33. 安装记录必须足以支持后续卸载。
34. 卸载只移除该配置包安装进去的内容。
35. 卸载不是整项目回滚。
36. 若某项内容不是本工具安装进去的，卸载时不得删除。
37. 若卸载时检测到目标内容已被用户手动改动且无法安全识别，系统必须阻止卸载并提示人工处理。
38. 用户修改收藏根目录后，系统必须自动迁移旧数据到新地址。
39. 迁移成功前不得删除旧数据。
40. 迁移失败时必须保留旧数据并提示失败原因。

# 9. 异常与边界

1. 本机未安装 Claude Code 时：
   1. 表单制作仍可使用。
   2. AI 生成功能必须提示不可用原因。
2. Claude Code 调用失败时：
   1. 不得生成脏数据。
   2. 必须提示失败原因或失败状态。
3. AI 返回内容格式不合法时：
   1. 系统不得自动保存。
   2. 必须提示用户修正。
4. 用户输入的项目根目录不存在时：
   1. 禁止安装。
   2. 必须提示目录无效。
5. 目标目录或目标文件无写权限时：
   1. 禁止安装。
   2. 必须提示权限不足。
6. 收藏根目录不存在时：
   1. 系统应允许创建。
   2. 创建失败时必须提示。
7. 收藏根目录无写权限时：
   1. 禁止保存。
   2. 必须提示路径不可写。
8. 用户修改收藏根目录且迁移失败时：
   1. 不得进入新旧数据混用状态。
   2. 必须保留旧数据。
9. 配置包包含多个对象时：
   1. 必须分别处理每个对象的目标位置。
   2. 不得只做包级笼统安装。
10. 某一对象校验失败时：
    1. 整个配置包不得保存成功。
    2. 必须明确指出失败对象。
11. 安装过程中某个对象合并失败时：
    1. 系统必须提示失败对象。
    2. 不得伪装成全部安装成功。
12. 卸载时若安装记录不存在：
    1. 禁止卸载。
    2. 必须提示无法识别来源。
13. 卸载时若目标内容已被手工改动：
    1. 禁止直接删除不确定内容。
    2. 必须提示人工处理。
14. 附加脚本缺失时：
    1. 禁止安装依赖该脚本的配置包。
    2. 必须提示缺失项。

# 11. 非功能要求

1. 一期运行平台优先支持 Windows 11。
2. 用户双击 `.exe` 后，浏览器页面应在 5 秒内打开。
3. 在收藏数量不超过 500 个配置包时：
   1. 列表打开时间应不超过 2 秒。
   2. 搜索结果响应时间应不超过 1 秒。
4. 文本编辑后，预览刷新时间应不超过 300 毫秒。
5. 单个配置包保存时间应不超过 1 秒，前提是包体积不超过 1 MB。
6. 单次安装操作应在 3 秒内完成，前提是涉及文件总大小不超过 5 MB。
7. 单次卸载操作应在 3 秒内完成，前提是涉及文件总大小不超过 5 MB。
8. 所有收藏数据默认保存在本地磁盘。
9. 除 AI 生成功能外，其余核心功能应可离线使用。
10. 系统异常退出后，已保存配置包与安装记录不得丢失。
11. 错误提示必须明确指出失败原因或失败对象。

# 12. 依赖与约束

1. AI 生成功能依赖本机可用的 Claude Code。
2. AI 生成功能依赖用户本机已有可用的 Claude Code 运行环境。
3. 安装功能依赖目标目录可写。
4. 工具本身不负责登录 Claude 账号。
5. 工具本身不负责初始化目标项目。
6. 工具只安装到 Claude Code 约定的标准目录。
7. 默认收藏根目录为 `D:\Programming_tools\.ccworkflow`。
8. 用户修改收藏根目录后，系统必须迁移旧数据。
9. 一期不要求兼容企业级集中策略或受管设置。

# 13. 待确认问题

1. 无。

# 14. 验收标准

1. 双击 `.exe` 后，系统可自动打开浏览器页面。
2. 用户可新建配置包，并保存到默认目录 `D:\Programming_tools\.ccworkflow`。
3. 用户可在设置中修改收藏根目录，系统会自动迁移旧数据，并在迁移后成功保存新的配置包。
4. 用户可通过表单新建至少 1 个 skill 对象并保存。
5. 用户可通过表单新建至少 1 个 hook 对象并保存。
6. 用户可通过表单新建至少 1 个 MCP 对象并保存。
7. 用户可在一个配置包中同时保存多个对象。
8. 用户可在一个配置包中同时保存不同类型对象。
9. 用户输入自然语言描述后，系统可调用本机 Claude Code 生成配置草稿。
10. AI 生成结果在用户确认前不会自动保存。
11. 用户可编辑 AI 草稿并再次保存。
12. 用户可在列表页按类型查看收藏。
13. 用户可按关键字搜索收藏。
14. 用户可按标签筛选收藏。
15. 用户选择“项目级安装”时，系统要求输入项目根目录。
16. 用户选择“用户全局安装”时，系统不要求输入项目根目录。
17. 目标文件不存在时，系统会自动创建后再安装。
18. 目标文件已存在时，系统会执行合并安装，而不是整文件覆盖。
19. 同名 skill、hook 或 MCP 存在时，系统会提示用户选择替换、跳过或取消安装。
20. 用户可成功将 skill 安装到指定项目。
21. 用户可成功将 hook 安装到指定项目。
22. 用户可成功将 MCP 安装到指定项目。
23. 用户可成功将配置包安装到用户全局目录。
24. 当配置包依赖附加脚本时，安装结果会包含这些脚本。
25. 当附加脚本缺失时，系统会阻止安装并提示原因。
26. 系统会为每次安装生成安装记录。
27. 用户可卸载某个已安装配置包。
28. 卸载时，系统只移除该配置包安装进去的内容。
29. 卸载不会删除非本工具安装的内容。
30. 当目标内容已被手工修改且系统无法安全识别时，系统会阻止卸载并提示人工处理。
31. 当 Claude Code 不可用时，系统仍可正常使用表单制作、收藏、检索功能。
