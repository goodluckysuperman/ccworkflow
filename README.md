# ccworkflow

一个基于 Python 3.11 的本地工具，用于制作、收藏、检索、安装和卸载 Claude Code 配置包。

当前一期已完成的配置类型：
- skill
- hook
- MCP

## 1. 项目目标

本项目用于解决 Claude Code 配置复用成本高、手工复制容易出错的问题。

一期重点解决：
1. 配置包制作
2. 配置包收藏
3. 配置包检索
4. 配置包安装
5. 配置包卸载
6. 本地打包交付

## 2. 一期功能范围

已实现能力：
1. 表单创建配置包
2. AI 生成配置草稿
3. 编辑并保存配置包
4. 多对象、多类型混合包
5. 按类型、关键字、标签检索
6. 项目级安装
7. 用户全局安装
8. 安装预览
9. 冲突检测与处理
10. 安装记录查询
11. 按记录安全卸载
12. 收藏根目录迁移
13. Windows 可执行文件打包

不在一期范围内：
1. 云同步
2. 多人协作
3. subagents 管理
4. rules / `CLAUDE.md` 管理
5. 项目配置实时监听
6. 任意历史回滚

## 5. 本地开发

### 5.1 安装依赖

```bash
uv sync
```

### 5.2 启动项目

```bash
uv run python main.py
```

默认行为：
1. 初始化收藏根目录
2. 启动本地 HTTP 服务
3. 自动打开浏览器页面

### 5.3 启动参数

可通过环境变量控制启动行为：

- `CCWORKFLOW_HOST`：监听地址，默认 `127.0.0.1`
- `CCWORKFLOW_PORT`：监听端口，不传则自动分配
- `CCWORKFLOW_OPEN_BROWSER`：是否自动打开浏览器，`0` 为关闭
- `CCWORKFLOW_RUN_SERVER`：是否真正启动服务，`0` 为仅执行启动准备逻辑

示例：

```bash
CCWORKFLOW_OPEN_BROWSER=0 CCWORKFLOW_PORT=8765 uv run python main.py
```

## 6. 使用方式

### 6.1 创建配置包

1. 打开“新建配置包”页
2. 填写名称、简介、标签
3. 添加一个或多个对象
4. 可选添加附加脚本
5. 保存配置包

### 6.2 AI 生成草稿

1. 打开“AI 生成草稿”页
2. 输入名称、简介、标签、目标类型和需求描述
3. 生成草稿
4. 点击“继续编辑”进入新建页
5. 调整后保存

说明：
- AI 草稿不会自动保存
- 若本机 Claude Code 不可用，AI 页面会提示不可用，但其他功能仍可使用

### 6.3 检索配置包

列表页支持：
1. 按类型筛选
2. 按关键字搜索
3. 按标签筛选

### 6.4 安装配置包

在配置包详情页可执行：
1. 安装预览
2. 冲突检查
3. 执行安装

支持两种范围：
1. 项目级安装
2. 用户全局安装

### 6.5 卸载配置包

在安装记录页可执行：
1. 查看记录详情
2. 按记录卸载

卸载规则：
1. 只移除本工具安装的内容
2. 若目标内容已被手工修改且无法安全识别，则阻止卸载

### 6.6 修改收藏根目录

在设置页可执行：
1. 更新普通设置
2. 迁移收藏根目录

迁移规则：
1. 会复制旧数据到新目录
2. 成功后切换到新目录
3. 迁移失败时保留旧目录
